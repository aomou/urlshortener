# Django URL Shortener

A URL shortener service built with Django, featuring OAuth login and click tracking.

[![Python](https://img.shields.io/badge/Python-3.13-blue.svg)](https://www.python.org/)
[![Django](https://img.shields.io/badge/Django-6.0-green.svg)](https://www.djangoproject.com/)
[![PostgreSQL](https://img.shields.io/badge/PostgreSQL-Database-blue.svg)](https://www.postgresql.org/)

## ✨ Key Features

- Create and manage short URLs per user
- Google OAuth login (no local password authentication)
- Enable/disable URLs for temporary deactivation
- Click tracking with basic statistics (total clicks)
- Filter and sort URLs by status and creation date
- Try as guest mode (24h temporary account with usage limits)

## 🏗️ Architecture

### Service Layer Pattern

```
┌─────────────┐
│   Views     │  ← HTTP layer: handles requests/responses, auth checks
└──────┬──────┘
       │
┌──────▼──────┐
│  Services   │  ← Business logic: URL creation, analytics, validation
└──────┬──────┘
       │
┌──────▼──────┐
│   Models    │  ← Data layer: schema definition only
└─────────────┘
```

**Key Services:**
- `URLService`: Short URL creation, retrieval, filtering, and toggling
- `AnalyticsService`: Click recording and statistics

### Data Models

`URLModel`
- Stores the original URL and its generated short code
- Links each short URL to its owner (`user`) for access control
- Prevents duplicates per user via a `(user, original_url)` unique constraint
- `is_active` field for soft deactivation

`ClickLog`
- Records a click event for a short URL (timestamp, IP, and user agent metadata)
- Enables per-URL click counts while presenting privacy-safe values (anonymized IP)
- Browser/OS/Device parsed via django-user-agents

## 🚀 Setup

### Prerequisites

- Python 3.13+
- PostgreSQL
- uv (Python package manager)

### Installation

1. Clone and install dependencies
   ```bash
   git clone <repository-url>
   cd urlshortener
   uv sync
   ```

2. Configure environment variables
   ```bash
   cp .env.example .env
   # Edit .env and update DATABASE_URL and SECRET_KEY
   ```

3. Setup database
   ```bash
   uv run python manage.py migrate
   uv run python manage.py createsuperuser  # Optional: only needed for Django Admin access
   ```

4. Run development server
   ```bash
   uv run python manage.py runserver
   ```

Visit `http://127.0.0.1:8000` to see the application.

**Note:** This project uses `127.0.0.1` instead of `localhost` for better OAuth compatibility. Some systems have DNS resolution issues with `localhost` that can cause OAuth callback failures.

### OAuth Setup

For OAuth login functionality:
1. Get credentials from [Google Cloud Console](https://console.cloud.google.com/)
2. Configure redirect URI in Google Cloud Console: `http://127.0.0.1:8000/accounts/google/login/callback/`
3. Add credentials to `.env` (see `.env.example` for format)

## 🌐 Deployment

Configured for [Render](https://render.com/) deployment:

1. Create PostgreSQL database and Web Service on Render
2. Set environment variables:
   - `DATABASE_URL` (auto-provided by Render)
   - `SECRET_KEY`
   - `RENDER_EXTERNAL_HOSTNAME` (auto-provided by Render)
   - OAuth credentials (optional, for social login)
3. Configure deployment:
   - Build Command: `./build.sh`
   - Start Command: `gunicorn core.wsgi:application`
4. Update OAuth redirect URIs in provider consoles to production domain

## 📝 Key Implementation Details

### Sqids URL Encoding

Short codes are generated by encoding `[user_id, url_id]`:
```python
sqids = Sqids(min_length=6)
short_code = sqids.encode([user.id, url_obj.id])
```

**Benefits:**
- Reversible encoding (decode back to IDs)
- User isolation (different codes for same URL)
- No collision (based on database IDs)

### IP Anonymization

- **Storage**: Full IP in database for audit trail
- **Display**: Anonymized in UI (192.168.1.100 → 192.168.1.0)
- **Compliance**: Privacy-first approach

### 302 vs 301 Redirects

Using **302 Temporary Redirect**:
- Prevents browser caching
- Ensures all clicks go through server
- Enables accurate analytics
